# 視覺與微互動準則

> 用途：定義振動監測儀表板的深色視覺風格、Design Tokens 色票、CSS 優先的動畫與微互動，讓畫面專業、克制、不吃效能。

技術棧前提：CSS（無 Tailwind）+ CSS 變數做 Design Tokens + 原生 CSS 動畫優先 + ECharts。

---

## 1. 視覺基調：工業風深色儀表板

參考 FineReport 類儀表板的深色工業風格，適合長時間盯看的監控場景：

- **深色底、低對比格線**，數值與狀態是主角，介面退到背景。
- **色彩當「訊號」而非裝飾**：平時整個畫面幾乎只有深藍灰，唯一的常駐彩色是主色（游標 / 選中 / 趨勢線）。綠 / 黃 / 紅只用在**狀態燈**與**越過門檻的資料點 / 門檻線**。
- 留白與對齊比花俏裝飾重要。卡片、間距、字級全部走同一套 token。

---

## 2. Design Tokens（CSS 變數集中管理）

絕對禁止在程式碼中四處 hardcode 色碼或間距。所有色彩 / 間距 / 動畫曲線集中在 `:root`，元件一律引用變數。

```css
:root {
  /* 色彩 ─ 深色工業儀表板（克制用色） */
  --bg: #0E1726;          /* 頁面底：深藍工業 */
  --surface: #16243B;     /* 卡片 / 面板 */
  --surface-2: #1E3050;   /* 內層 / hover */
  --text: #C7D2E0;        /* 主文字 */
  --text-dim: #7C8BA1;    /* 次要文字 */
  --grid: rgba(255,255,255,.06); /* 格線 / 分隔 */

  --accent: #22D3EE;      /* 主色：游標 / 選中 / 趨勢線（平時唯一彩色） */

  /* 狀態色 ─ 只用在狀態燈與越界資料點 / 門檻線 */
  --good: #22C55E;        /* 良好（綠） */
  --warning: #F59E0B;     /* 異常（琥珀） */
  --danger: #EF4444;      /* 危險（紅） */

  /* 間距 */
  --space-1: 4px;  --space-2: 8px;  --space-3: 16px;  --space-4: 24px;

  /* 圓角 / 字級 */
  --radius: 8px;
  --font-value: 28px;     /* 振動數值 */

  /* 動畫曲線（統一引用） */
  --ease: cubic-bezier(.25,.8,.25,1);
  --transition: .3s var(--ease);
}
```

色彩語意對應 ISO 10816 分區：

| Token | 狀態 | 用在哪 |
|-------|------|--------|
| `--good` | 良好 | 狀態燈、趨勢圖正常區 |
| `--warning` | 異常 | 狀態燈、越過警告門檻的資料點 / 門檻線 |
| `--danger` | 危險 | 狀態燈、越過危險門檻的資料點 / 門檻線 |
| `--accent` | 中性互動 | 游標 / axisPointer、選中卡片邊框、趨勢線本體 |

ECharts 的 `color`、`lineStyle`、`markLine` 等設定也應讀同一套變數值（透過 `getComputedStyle(document.documentElement).getPropertyValue('--accent')`），避免圖表配色與 CSS 不一致。

## 3. CSS 優先動畫（只動 transform / opacity）

- **能用 CSS 就不用 JS**：hover、淡入淡出、選中縮放、狀態高亮，一律用 CSS `transition` / `animation`（GPU 加速、效能最好）。不引入任何 JS 動畫套件。
- **只動 `transform` 與 `opacity`**：這兩個屬性不觸發 Reflow / Repaint。**嚴禁**動畫頻繁改變 `width`、`height`、`margin`、`top` / `left`（這些會逼瀏覽器重排，儀表板有 20 張卡片時特別明顯）。
- 動畫曲線統一引用 `--ease` / `--transition`，維持一致手感。

```css
.device-card {
  transition: transform var(--transition), box-shadow var(--transition);
}
.device-card:hover { transform: translateY(-2px); } /* 不是改 margin */

.device-card--selected { box-shadow: 0 0 0 2px var(--accent); }
```

## 4. 骨架屏（Skeleton）

資料回來前先顯示排版輪廓，避免空白與版面跳動。純 CSS shimmer，只動 `transform`（位移漸層），不動版面。

```css
.skeleton {
  background: var(--surface);
  border-radius: var(--radius);
  position: relative;
  overflow: hidden;
}
.skeleton::after {
  content: "";
  position: absolute; inset: 0;
  background: linear-gradient(90deg, transparent, var(--surface-2), transparent);
  transform: translateX(-100%);
  animation: shimmer 1.4s infinite;
}
@keyframes shimmer { 100% { transform: translateX(100%); } }
```

## 5. 狀態變化的脈衝 / 高亮微互動

當設備數值由 SignalR 更新、或狀態跨越門檻時，給一個短暫、克制的視覺反饋，讓使用者注意到「這張卡剛剛變了」，但不打擾整個畫面。

```css
/* 數值剛更新：短暫高亮一下背景 */
@keyframes flash {
  0%   { background: var(--surface-2); }
  100% { background: var(--surface); }
}
.device-card--updated { animation: flash .6s var(--ease); }

/* 危險狀態：狀態燈緩慢脈衝（只動 opacity） */
@keyframes pulse { 0%,100% { opacity: 1; } 50% { opacity: .4; } }
.status--danger .status__dot { animation: pulse 1.2s infinite; }
```

```js
// 渲染模組差異更新後，加一次性高亮 class
function flashUpdated($card) {
  $card.removeClass('device-card--updated');
  void $card[0].offsetWidth;        // 重置動畫
  $card.addClass('device-card--updated');
}
```

原則：脈衝只給「危險」這種需要持續注意的狀態；一般數值更新只閃一下就停，不要整面卡片都在動。

## 6. 跨元件動態連動

儀表板各部件應協調呼應，靠 `AppState` + 自訂事件連動（見 01），而非各自為政：

- **選中卡片 → 趨勢圖**：點某張卡片，卡片加選中邊框（`--accent`），同時趨勢圖載入該設備資料。
- **游標連動**：ECharts 趨勢圖的 `axisPointer` / `tooltip` 跟著滑鼠移動顯示對應時間點的數值（游標移動需 throttle，見 04）。
- **門檻線**：趨勢圖用 `markLine` 畫出警告 / 危險門檻線，顏色用 `--warning` / `--danger`，越線的資料點才上色。

```js
// 點卡片 → 更新選中狀態（事件驅動，渲染與圖表自動跟上）
$('#device-grid').on('click', '.btn-detail', function () {
  AppState.set('selectedId', $(this).data('device-id'));
});
```

## 7. 微互動小規範

- **Toast**：螢幕角落非阻擋式提示，3~5 秒自動淡出，不擋使用者操作。tone 對應 `--good` / `--warning` / `--danger` / `--accent`。
- **按鈕狀態**：觸發請求時立即 `disabled` 並顯示「處理中…」，避免連點；完成後解除。
- **focus 樣式**：互動元件保留可見的 focus 樣式（用 `--accent`），鍵盤操作看得到焦點。

## 8. 視覺檢查清單

- [ ] 綠黃紅只用在狀態燈與越界資料點 / 門檻線，平時畫面克制
- [ ] 動畫只改 `transform` / `opacity`，未動 width/height/margin/top/left
- [ ] 載入有骨架屏，數值更新有一次性高亮，危險狀態有脈衝
