# 前端資安準則

> 用途：定義本儀表板必要的前端資安防護：XSS 防禦、antiforgery / CSRF Token、CSP 與安全標頭、機敏資訊不放 URL。

技術棧前提：ASP.NET Core 9 MVC（Razor）+ jQuery + IIS（web.config）+ SignalR（WebSocket）。即使是內部系統，這些仍是基本盤。

---

## 1. XSS 防禦（跨站腳本攻擊）

攻擊者在頁面注入惡意 JavaScript。本專案會顯示設備名稱、備註等可能來自資料庫 / 外部的字串，必須防範。

### 1.1 jQuery：一律用 `.text()`，不用 `.html()`

注入使用者 / 外部資料時，**用 `.text()`**（瀏覽器自動 HTML-encode），**不要用 `.html()`**（會直接執行其中的標籤 / 腳本）。

```js
// ✅ 安全：.text() 自動轉義
$('.device-card__name').text(device.name);

// ❌ 危險：device.name 若含 <img src=x onerror=alert(1)> 會被執行
$('.device-card__name').html(device.name);
```

若**必須**插入 HTML（如富文本），先消毒再插入：

```js
// 必要時才用，且務必先消毒（引入 DOMPurify）
$('#notes').html(DOMPurify.sanitize(rawHtml));
```

組裝屬性 / URL 時也要小心：用 `.attr()` 設值（會轉義），避免用字串拼接 HTML。

### 1.2 Razor：預設會 encode，留意 `Html.Raw`

- Razor 的 `@Model.Name` **預設會 HTML-encode**，是安全的。
- **`@Html.Raw(...)` 會原樣輸出，是 XSS 風險點**。只在輸出「自己產生且已知安全」的內容（如序列化的初始資料 JSON）時使用，**絕不對使用者 / 外部輸入用 `Html.Raw`**。

```cshtml
@* ✅ 安全：自動 encode *@
<span>@device.Name</span>

@* ⚠️ 風險：只能用在可信內容（如自家序列化的 JSON），不可用於使用者輸入 *@
<script id="initial-data" type="application/json">@Html.Raw(Model.DevicesJson)</script>
```

內嵌 JSON 時，序列化端應確保正確轉義 `<`、`>`、`&`，避免 `</script>` 提前關閉腳本標籤。

## 2. Antiforgery / CSRF Token

防止跨站請求偽造。使用 ASP.NET Core 內建 antiforgery 機制。

### 2.1 Razor 產出 token

在頁面放一個 antiforgery token（表單用 `@Html.AntiForgeryToken()`，或直接放隱藏欄位供 AJAX 讀取）：

```cshtml
@Html.AntiForgeryToken()
@* 產出： <input name="__RequestVerificationToken" type="hidden" value="..."> *@
```

### 2.2 前端 AJAX 帶上 token

統一在 `api.js`（見 02）的 header 帶 `RequestVerificationToken`，所有寫入請求（POST / PUT / DELETE）自動附上：

```js
headers: { 'RequestVerificationToken': $('input[name="__RequestVerificationToken"]').val() }
```

### 2.3 後端驗證

- 寫入用的 Controller / Action 加 `[ValidateAntiForgeryToken]`（或全域 `AutoValidateAntiforgeryToken`）。
- 在 `Program.cs` 設定讀 header 的名稱：

```csharp
builder.Services.AddAntiforgery(o => o.HeaderName = "RequestVerificationToken");
```

## 3. CSP 與安全標頭

CSP（內容安全政策）是攔截 XSS 的最後防線。由伺服器派發 HTTP Header（優於 `<meta>`，因 `<meta>` 無法設 `frame-ancestors`）。

### 3.1 本專案適用的 CSP

允許 `self`；若 ECharts / jQuery 走 CDN 需列白名單；`connect-src` 要放行 SignalR 的 `ws` / `wss`。

```
default-src 'self';
script-src 'self' https://cdn.jsdelivr.net;            /* 若 ECharts/jQuery 走 CDN 才加白名單，否則只留 'self' */
style-src 'self' 'unsafe-inline';                       /* 內嵌樣式需要時才加 unsafe-inline */
img-src 'self' data:;
font-src 'self';
connect-src 'self' ws: wss:;                            /* SignalR WebSocket；可收斂成自家網域 wss://yourhost */
frame-ancestors 'none';                                 /* 禁止被嵌入 iframe，防 Clickjacking */
base-uri 'self';
form-action 'self';
```

要點：
- 能不開 CDN 就把 ECharts / jQuery 放本機 `self`，`script-src` 就不必開白名單，最安全。
- `connect-src` **必須包含 `ws:` / `wss:`**，否則 SignalR WebSocket 會被 CSP 擋下。盡量收斂成自家網域（如 `wss://yourhost`）。
- 禁用 `'unsafe-eval'`；`'unsafe-inline'` 只在無法避免的內嵌樣式 / 腳本時才開，能用 nonce 更佳。

### 3.2 派發方式 A：IIS web.config

```xml
<system.webServer>
  <httpProtocol>
    <customHeaders>
      <add name="Content-Security-Policy"
           value="default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data:; connect-src 'self' ws: wss:; frame-ancestors 'none'; base-uri 'self'; form-action 'self';" />
      <add name="X-Content-Type-Options" value="nosniff" />
      <add name="X-Frame-Options" value="DENY" />
      <add name="Referrer-Policy" value="no-referrer" />
    </customHeaders>
  </httpProtocol>
</system.webServer>
```

### 3.3 派發方式 B：ASP.NET Core middleware

需要動態（如每請求產 nonce）時，用 middleware 派發：

```csharp
// Program.cs
app.Use(async (context, next) =>
{
    var h = context.Response.Headers;
    h["Content-Security-Policy"] =
        "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; " +
        "img-src 'self' data:; connect-src 'self' ws: wss:; " +
        "frame-ancestors 'none'; base-uri 'self'; form-action 'self';";
    h["X-Content-Type-Options"] = "nosniff";
    h["X-Frame-Options"] = "DENY";
    h["Referrer-Policy"] = "no-referrer";
    await next();
});
```

兩者擇一即可（避免重複設定造成衝突）。

## 4. 機敏資訊不放 URL

- 任何具識別性的機敏資料（金鑰、token、密碼、個資）**絕不可出現在 URL 查詢字串**（`?token=...`）。URL 會被瀏覽紀錄、IIS log、代理伺服器記錄下來。
- 寫入 / 機敏操作走 POST body，不走 query string。
- 內部系統雖無 SEO 顧慮，但設備代號等若屬敏感，也避免曝在可被輕易書籤 / 分享的網址。

---

## 5. 資安檢查清單

- [ ] jQuery 注入外部資料一律 `.text()`，必要的 HTML 先 `DOMPurify.sanitize`
- [ ] Razor 預設 encode；`Html.Raw` 僅用於可信內容，未用於使用者輸入
- [ ] 寫入 API 帶 antiforgery token（header），後端有驗證
- [ ] CSP 由 web.config 或 middleware 派發，`connect-src` 含 `ws:` / `wss:`
- [ ] 加上 `X-Content-Type-Options` / `X-Frame-Options` / `Referrer-Policy`
- [ ] 機敏資訊不出現在 URL query string
