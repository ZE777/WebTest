# 效能準則

> 用途：定義儀表板必須執行的效能手段，重點在 ECharts 延遲載入、游標 / resize 節流、卡片差異更新、避免不必要的重繪重排。

技術棧前提：原生 JS + jQuery + ECharts。20 台設備、即時更新、可移動游標趨勢圖。

---

## 1. ECharts 延遲載入與 instance 管理

ECharts 是大型圖表庫，不該在首屏就載入與初始化。

- **延遲載入函式庫**：首屏只需設備卡片，不需圖表。使用者**點某台設備看明細時**才動態載入 ECharts 並初始化圖表 instance。
- **instance 重用，不重複建立**：同一個圖表容器只 `echarts.init` 一次，之後切換設備只 `setOption` 換資料，不要每次 `dispose` 再 `init`。
- **離開 / 關閉面板時 `dispose`**：明細面板關閉後釋放 instance，避免記憶體累積與殘留的 resize 監聽。

```js
// trendChart.js — 延遲載入 + instance 重用
const TrendChart = (function () {
  let chart = null;        // 單一 instance
  let echartsLib = null;   // 延遲載入的函式庫

  // 動態載入 ECharts（首次點明細才下載）
  function ensureLib() {
    if (echartsLib) return $.Deferred().resolve(echartsLib).promise();
    return $.getScript('/lib/echarts/echarts.min.js').then(function () {
      echartsLib = window.echarts;
      return echartsLib;
    });
  }

  function load(deviceId) {
    ensureLib().then(function (echarts) {
      if (!chart) {
        chart = echarts.init(document.getElementById('trend-chart'));
        bindResize();              // 只綁一次
      }
      api.get('/api/devices/' + deviceId + '/trend')
         .done(data => chart.setOption(buildOption(data))); // 重用 instance，只換資料
    });
  }

  function destroy() {
    if (chart) { chart.dispose(); chart = null; }
    unbindResize();
  }

  return { load, destroy };
})();
```

## 2. 游標 / resize 的 throttle / debounce

高頻事件不可每次都重算或重繪，否則畫面卡頓。

- **游標移動（axisPointer / tooltip）→ throttle**：限制更新頻率（如每 50ms 最多一次），手感順又不爆 CPU。ECharts 內建 tooltip 已有部分節流，但自訂的滑鼠連動邏輯要自己包 throttle。
- **視窗 resize → debounce**：resize 連續觸發，等使用者停下來（如 200ms）才呼叫一次 `chart.resize()`，避免連續重排。

```js
// 簡易 throttle / debounce（不需引入 lodash）
function throttle(fn, wait) {
  let last = 0;
  return function () {
    const now = Date.now();
    if (now - last >= wait) { last = now; fn.apply(this, arguments); }
  };
}
function debounce(fn, wait) {
  let t;
  return function () {
    clearTimeout(t);
    const ctx = this, args = arguments;
    t = setTimeout(() => fn.apply(ctx, args), wait);
  };
}

// resize：debounce 後才 resize 圖表
const onResize = debounce(() => { if (chart) chart.resize(); }, 200);
function bindResize()   { $(window).on('resize.trend', onResize); }
function unbindResize() { $(window).off('resize.trend'); }

// 自訂游標連動：throttle
const onAxisMove = throttle(function (params) { /* 更新外部讀數面板 */ }, 50);
```

## 3. 卡片差異更新（避免全量重建 DOM）

20 張卡片不需要虛擬列表，但**絕不可每次推播就整批重建 DOM**（會閃爍、丟失 hover / 選中狀態、浪費效能）。改成只更新有變動的卡片、只改變動的欄位。

- 首次渲染建立全部卡片，每張掛 `data-device-id`。
- SignalR 推來單台更新時，找到對應卡片，**只改變動的欄位**（數值、狀態 class），不重建整張卡。
- 比對前後資料，數值沒變就不碰 DOM（避免無謂 Repaint 與動畫）。

```js
// 渲染模組：差異更新單張卡片
const DeviceRenderer = (function () {
  function patch(device) {
    const $card = $('#device-' + device.id);
    if (!$card.length) return;

    // 只在值真的變了才更新 DOM
    const $value = $card.find('.device-card__value');
    if ($value.data('raw') !== device.value) {
      $value.data('raw', device.value).text(device.value); // .text 防 XSS，見 05
      flashUpdated($card);                                  // 一次性高亮，見 03
    }

    // 狀態 class 變了才切換
    const statusClass = 'status--' + device.status;
    const $status = $card.find('.status');
    if (!$status.hasClass(statusClass)) {
      $status.removeClass('status--good status--warning status--danger')
             .addClass(statusClass)
             .text(device.statusLabel);
    }
  }
  return { patch };
})();
```

要點：用 `id` / `data-` 精準定位節點；先比對再寫入；用 `.text()` 而非 `.html()`（同時防 XSS）。

## 4. 圖檔 / 靜態資源快取

- **瀏覽器快取**：圖示、JS、CSS 等靜態檔交由 IIS 設定快取（`Cache-Control` / `ETag`），未變動的資源回 `304 Not Modified`，不重抓。
- **趨勢資料快取（可選）**：若同一設備短時間重複查詢，後端可回 `ETag`，前端帶 `If-None-Match` 拿 304，省頻寬。
- **ECharts 等大型 lib** 放固定路徑並設長快取，延遲載入後瀏覽器會快取，第二次點明細不再下載。

```xml
<!-- web.config：靜態資源快取（30 天） -->
<staticContent>
  <clientCache cacheControlMode="UseMaxAge" cacheControlMaxAge="30.00:00:00" />
</staticContent>
```

## 5. 避免不必要的重繪 / 重排（Reflow / Repaint）

- **批次 DOM 操作**：一次更新多張卡片時，先在記憶體裡組好（DocumentFragment / 字串拼接）再一次塞進 DOM，不要在迴圈裡逐次 append。
- **讀寫分離**：避免「讀 `offsetWidth` → 寫樣式 → 又讀 → 又寫」交錯，會觸發強制同步重排（layout thrashing）。先全部讀、再全部寫。
- **動畫只走 transform / opacity**（見 03）。
- **`aria-live` 區域**勿放整頁，只標在會變動的數值節點，避免報讀器與瀏覽器反覆處理大區塊。
