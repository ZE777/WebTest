# 設計知識紀錄(docs)

這個資料夾記錄本專案「**怎麼設計、考量點是什麼、怎麼實作**」的決策過程,
作為面試報告的素材。每份文件都力求**詳細但不複雜、可快速抓重點**。

## 專案一句話
用 ASP.NET Core MVC(掛 IIS)做一個**振動狀態監測儀表板**:動態顯示 10~20 台設備的
振動數值與狀態,檔案更新時即時刷新,點選設備可看可移動游標的趨勢圖。

## 重點速記(報告時可直接講的 3 句話)
1. **架構**:用 `IMachineDataSource` 介面把資料來源抽象化,JSON 檔與 MS-SQL 是兩個可切換實作 → 展示可維護性與資料庫能力。
2. **即時**:用 `FileSystemWatcher` 偵測檔案變更 → 透過 SignalR **只推變動的那一台**,而非全頁輪詢 → 省效能、低延遲。
3. **領域**:把題目對焦到公司本業「振動監測」,狀態對接 **ISO 10816** 分區,趨勢圖畫門檻線與分區色帶 → 看起來像真的監控軟體。
